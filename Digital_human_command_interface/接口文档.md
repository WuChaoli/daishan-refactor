# Streaming Chat FastAPI · API 接口文档

- **服务名**：Streaming Chat FastAPI  
- **版本**：1.0.0  
- **Base URL**：`http://172.16.11.60:11029`  
- **OpenAPI/Swagger**：`GET /docs`  
- **编码**：UTF-8  


## 全局约定
1. **鉴权**：无（服务端自行转发至上游 SSE 接口）。  
2. **CORS**：允许所有来源（开发阶段 `*`；生产建议收敛域名）。  
3. **通用请求头**：  
   - `Content-Type: application/json`（除健康检查外）  
4. **流式响应头**：  
   - `Content-Type: text/event-stream`  
   - `X-Content-Type-Options: nosniff`  
   - `Cache-Control: no-cache`  
   - `Connection: keep-alive`  

> 说明：本服务将上游 SSE 解析后，以**纯文本分片**返回（不是标准 `data:` 帧）。请用 `fetch + ReadableStream` 或等价方式消费。  


## 1) 流式聊天（打字机效果）
**请求方法与路径**：`POST /api/stream-chat`  

功能说明：将用户输入转发至上游聊天接口，按“打字机效果”逐块返回文本。


### 请求体（JSON）
| 字段           | 类型     | 必填 | 说明                              |
| ------------ | ------ | -- | ------------------------------- |
| `text_input` | string | 是  | 用户输入内容，不能为空或纯空白                 |
| `user_id`    | string | 否  | 用户标识；默认 `"fastapi_client_user"` |

#### 请求体示例
```json
{
  "text_input": "介绍园区",
  "user_id": "user_001"
}
```


### 成功响应
- **HTTP 状态码**：200  
- **Content-Type**：`text/event-stream`  
- **响应体（文本流，非标准 SSE）**：按以下顺序包含内容  
  1. 开始标记：`【开始流式输出（打字机效果）】\n`  
  2. 内容分片：以约 3 字符为一片逐块输出  
  3. 结束标记：`\n\n[提示] 流式输出已完成\n`  

> 过程中可能出现行内提示：  
> `"[警告] 解析流式数据失败: ..."`、`"[错误] API流式处理错误: ..."` 等。  


### 流式响应示例
#### 请求
```http
POST http://172.16.11.60:11029/api/stream-chat
Body: {"text_input":"打开产业分析驾驶舱","user_id":"test_user_001"}
```

#### 响应（`text/event-stream`，按纯文本分片返回，非标准 SSE 帧）
```text
【开始流式输出（打字机效果）】{  "status": "success",  "english_command": "global+iaTopic",  "feedback_description": ""}[提示] 流式输出已完成
```

> 说明  
> 1. 开头有固定开始标记：`【开始流式输出（打字机效果）】`  
> 2. 中间是模型产出的**正文**（此处为一段 JSON 文本）  
> 3. 结尾有固定结束标记：`[提示] 流式输出已完成`  
> 4. 由于采用分片输出，以上三段可能并不总是分行显示，客户端需做**缓冲拼接**后再解析。
