## 提示词优化

### 优化内容

1. **明确判断规则**
   - 增加了详细的分类标准和示例
   - 明确了"完整名称"vs"占位符"的区别
   - 强调了位置查询关键词的识别

2. **关键改进点**
   - 添加了"动词不影响判断"的说明
   - 明确了"XX"、"某个"等占位符应归类为requirement
   - 强调了"最近的"、"附近"、"哪里有"等位置查询词汇
   - 区分了"泛指需求"和"具体资源"

3. **修复问题**
   - 修复了拼写错误: "resoure" → "resource"
   - 优化了提示词结构，使用分层标题和要点

### 优化后的提示词结构

```
【判断规则】
- Resource: 明确指定具体资源（完整名称）
- Requirement: 模糊需求（位置查询、占位符、泛指）

【关键判断点】
1. 具体完整名称 → resource
2. 位置查询词汇 → requirement
3. 占位符/泛指 → requirement
4. 动词不影响判断

【输出要求】
只返回 "resource" 或 "requirement"
```

## 关键发现

### 边界情况处理

1. **占位符识别**
   - "XX医院"、"XX污水处理厂" → requirement ✓
   - 系统能正确识别占位符并归类为模糊需求

2. **位置查询识别**
   - "最近的"、"附近"、"哪里有" → requirement ✓
   - 系统能准确识别位置相关查询

3. **动词影响**
   - "我要去东区人民医院" → resource ✓
   - "联系市中心消防站" → resource ✓
   - 动词不影响判断，关键看是否有完整名称

4. **泛指需求识别**
   - "需要专业救援人员" → requirement ✓
   - "需要救援" → requirement ✓
   - 系统能识别泛指修饰词
