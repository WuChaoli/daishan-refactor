## 背景

当前业务日志主要依赖树形文本缩进，在异步协程和多 worker 场景下，容易出现调用层级串线、上下文错位的问题，不利于快速定位请求路径与函数行为。

## 目标

新建独立项目 `LogTracer`，不修改现有项目代码，提供可复用的调用链观测能力，覆盖：

1. 函数调用链追踪（同步/异步）
2. 入参/出参摘要记录（脱敏、截断）
3. OpenTelemetry 追踪接入（Jaeger 可视化）
4. 多 worker 日志隔离
5. 异常链路观测与诊断

## 非目标

- 本阶段不改造既有 `src/log_decorator` 模块
- 本阶段不接入旧业务服务代码
- 本阶段不引入 Docker 作为必选运行模式（以 `./.venv` 本地运行为主）

## 范围约束

- 与旧日志实现完全隔离，不复用旧实现代码
- 配置优先级固定为：`.env` > `config.yaml` > `json`
- 代码组织遵循：`src/`、`test/`、`scripts/` 分离

## 成功标准

- 单请求可在 Jaeger 中完整查看调用链与耗时瀑布
- 任意函数可查看入参与出参摘要，敏感字段默认脱敏
- 多 worker 并发下可按 `trace_id` 稳定回放链路
- 异常链路可在 Trace 与日志间互相定位
- 关键模块测试覆盖率达到 80%+
