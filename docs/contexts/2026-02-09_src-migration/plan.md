# 代码项目统一迁移到 `src/` 计划（阶段1→阶段2）

## 1. 目标与范围

本计划按“先稳后彻底”执行：

- **阶段1（方案1）**：物理迁移到 `src/`，同时保留根目录兼容入口/兼容壳，确保历史调用路径不立即失效。
- **阶段2（方案2）**：在阶段1稳定后，移除兼容壳并全量收敛引用到新路径。

已确认范围为 **A+B**：

- A：`Digital_human_command_interface`、`rag_stream`、根项目入口
- B：`DaiShanSQL`、`dify_sdk`、`ragflow_sdk`、`log_decorator`

## 2. 阶段1（兼容迁移）

### 2.1 执行步骤

1. 建立迁移基线（目录快照、入口清单、import 清单、可回滚点）。
2. 将上述代码项目迁入 `src/` 下统一管理。
3. 在根目录保留兼容层（启动脚本/导入壳/路径代理），保证旧命令可用。
4. 修正路径解析（`sys.path`、相对路径、配置文件路径、日志路径）。
5. 修正打包与安装配置（`setup.py` / `pyproject.toml` / `package_dir` / `find_packages`）。
6. 运行冒烟验证 + 关键测试。

### 2.2 验收标准（Gate A）

- 旧入口命令可运行（兼容壳生效）。
- 新路径入口可运行（`src/...`）。
- 关键测试通过（以受影响模块为主）。
- 无阻断级 import/path 错误。

## 3. 阶段2（彻底收敛）

### 3.1 执行步骤

1. 全仓替换对旧路径的引用（代码、脚本、测试、文档中的执行指令）。
2. 移除阶段1兼容壳与临时适配代码。
3. 清理冗余路径逻辑与遗留说明。
4. 执行全量回归（测试、核心启动链路、关键脚本）。
5. 更新文档与迁移说明，输出最终路径规范。

### 3.2 验收标准（Gate B）

- 不再依赖旧路径兼容壳。
- 所有代码项目均在 `src/` 下且可正常访问。
- 测试与核心运行链路通过。

## 4. 风险与应对

- **风险1：运行时导入断裂**
  - 应对：阶段1保留兼容壳 + 冒烟先行。
- **风险2：测试硬编码旧路径**
  - 应对：集中扫描并分批替换，先保证关键链路。
- **风险3：打包配置失配**
  - 应对：逐项目校验 `setup.py/pyproject`，必要时补充最小安装验证。

## 5. 交付物

- 阶段1交付：`src/` 迁移完成 + 兼容层 + 验证结果报告。
- 阶段2交付：兼容层移除 + 全路径收敛 + 文档更新 + 最终验证报告。
