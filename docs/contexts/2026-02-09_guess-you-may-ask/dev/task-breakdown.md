## 技术要点

### 依赖服务
- **IntentService**: 意图识别服务，提供process_query()方法
- **log_decorator**: 日志装饰器，用于记录接口调用

### 数据结构
```python
# IntentResult
{
    "type": int,  # 意图类型：0/1/2
    "query": str,  # 用户问题
    "results": [  # 查询结果列表
        {
            "question": str,  # 问题文本
            "similarity": float  # 相似度
        }
    ],
    "similarity": float,  # 最高相似度
    "database": str  # 数据库名称
}

# 响应格式
{
    "code": 0,  # 0成功，1失败
    "message": "成功",
    "data": [
        {"guess_your_question": "问题1"},
        {"guess_your_question": "问题2"},
        {"guess_your_question": "问题3"}
    ]
}
```

### 关键逻辑
1. **类别1（type=1）**: 返回固定的3个问题
2. **类别2（type=2）**: 提取results[1:4]的question字段
3. **其他类别**: 返回空列表
4. **错误处理**: 捕获异常并返回统一错误格式

## 风险点

### 技术风险
1. **IntentService异步调用**: 需要正确处理async/await
2. **数据格式变化**: results可能为空或格式不正确
3. **性能问题**: 意图识别可能耗时较长

### 缓解措施
1. 使用try-except捕获所有异常
2. 添加数据格式验证
3. 添加超时处理（如果需要）
4. 完善的单元测试和集成测试
