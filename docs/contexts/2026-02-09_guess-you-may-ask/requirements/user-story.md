# 猜你想问功能 - 用户故事

## 用户故事1：获取推荐问题

**作为** 应急管理系统的用户
**我想要** 在提问后看到相关的推荐问题
**以便于** 我能够发现更多有价值的信息，深入了解相关主题

### 场景描述

用户在使用应急管理系统时，提出了一个关于应急资源的问题。系统在返回答案的同时，还提供了3个相关的推荐问题，帮助用户进一步探索。

### 具体场景

#### 场景1：查询应急资源（类别1）

**前置条件**：
- 用户已登录系统
- 用户提出了一个关于应急资源的问题

**操作步骤**：
1. 用户输入问题："附近有哪些应急避难所？"
2. 前端调用 `/api/guess-questions` 接口
3. 系统识别意图类型为类别1
4. 系统返回3个固定的推荐问题

**预期结果**：
```json
{
  "code": 0,
  "message": "成功",
  "data": [
    {"guess_your_question": "如何查询应急资源？"},
    {"guess_your_question": "如何调度救援队伍？"},
    {"guess_your_question": "如何查看事故信息？"}
  ]
}
```

**用户体验**：
- 用户看到3个相关的推荐问题
- 用户可以点击推荐问题继续探索
- 推荐问题帮助用户发现更多功能

#### 场景2：查询数据库问题（类别2）

**前置条件**：
- 用户已登录系统
- 用户提出了一个数据库相关的问题

**操作步骤**：
1. 用户输入问题："岱山有多少家化工企业？"
2. 前端调用 `/api/guess-questions` 接口
3. 系统识别意图类型为类别2
4. 系统从意图识别结果中提取第2-4个相关问题

**预期结果**：
```json
{
  "code": 0,
  "message": "成功",
  "data": [
    {"guess_your_question": "岱山有多少家危化品企业？"},
    {"guess_your_question": "岱山化工园区的企业分布情况？"},
    {"guess_your_question": "岱山重大危险源有哪些？"}
  ]
}
```

**用户体验**：
- 用户看到3个与当前问题相关的推荐问题
- 推荐问题基于系统的知识库，具有较高的相关性
- 用户可以通过推荐问题深入了解相关数据

#### 场景3：错误处理

**前置条件**：
- 用户已登录系统
- 意图识别服务暂时不可用

**操作步骤**：
1. 用户输入问题："应急预案有哪些？"
2. 前端调用 `/api/guess-questions` 接口
3. 系统调用意图识别服务失败

**预期结果**：
```json
{
  "code": 1,
  "message": "意图识别服务暂时不可用",
  "data": []
}
```

**用户体验**：
- 用户看到友好的错误提示
- 系统不会崩溃或返回异常信息
- 用户可以继续使用其他功能

## 用户故事2：个性化推荐（后续扩展）

**作为** 应急管理系统的用户
**我想要** 根据我的历史查询记录获得个性化的推荐问题
**以便于** 我能够更快地找到我关心的信息

### 场景描述

系统记录用户的历史查询记录，分析用户的兴趣偏好，在推荐问题时优先推荐用户可能感兴趣的主题。

**注**：此功能为后续扩展，当前版本不实现。

## 用户价值

1. **提升探索效率**：用户无需自己思考下一步要问什么，系统主动提供相关问题
2. **发现隐藏功能**：通过推荐问题，用户可以发现系统的更多功能和数据
3. **增强交互体验**：智能推荐让系统更加主动和友好
4. **提高用户粘性**：用户在系统中停留更长时间，探索更多内容

## 用户反馈

预期用户反馈：
- "推荐的问题很有用，帮我发现了很多我不知道的功能"
- "不用自己想问题了，系统会主动提示"
- "推荐的问题和我的问题很相关，很智能"
