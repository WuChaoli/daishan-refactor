# 猜你想问功能需求文档

## 背景

当前系统在用户提问后，仅返回答案，缺少引导用户进一步探索的机制。为了提升用户体验和交互深度，需要在用户提问后，根据问题的意图类型，智能推荐相关的后续问题，帮助用户发现更多有价值的信息。

## 目标

实现"猜你想问"功能，在用户提问后，根据意图识别结果，智能推荐3个相关问题，引导用户进行深度探索，提升系统的交互体验和用户粘性。

## 功能描述

### 1. 接口设计

**接口路径**：`POST /api/guess-questions`

**输入参数**：
- `question` (string, 必填): 用户的问题文本

**返回格式**：
```json
{
  "code": 0,
  "message": "成功",
  "data": [
    {"guess_your_question": "问题1"},
    {"guess_your_question": "问题2"},
    {"guess_your_question": "问题3"}
  ]
}
```

### 2. 核心逻辑

#### 2.1 意图识别
调用 `IntentService.process_query(question, user_id)` 获取意图识别结果：
- `type`: 意图类型（0/1/2）
- `results`: 查询结果列表，每个元素包含 `question` 和 `similarity` 字段

#### 2.2 问题生成策略

**类别1（type=1）**：
- 返回固定的三个推荐问题
- 问题内容预留为模板，后续可配置修改
- 示例模板：
  - "如何查询应急资源？"
  - "如何调度救援队伍？"
  - "如何查看事故信息？"

**类别2（type=2）**：
- 从意图识别结果的 `results` 列表中提取第2-4个结果
- 提取每个结果的 `question` 字段
- 如果结果不足3个，返回实际可用的结果数量

**其他类别（type=0或其他）**：
- 返回空列表或默认推荐问题

### 3. 非功能需求

#### 3.1 日志记录
- 使用 `log_decorator` 装饰器记录接口调用
- 记录关键信息：
  - 用户问题
  - 意图识别结果（type、similarity）
  - 生成的推荐问题列表

#### 3.2 错误处理
- 捕获意图识别服务调用失败
- 捕获数据格式错误（results列表为空或格式不正确）
- 返回统一的错误响应格式：
```json
{
  "code": 1,
  "message": "错误信息",
  "data": []
}
```

#### 3.3 性能要求
- 接口响应时间 < 2秒
- 不影响主聊天接口的性能

## 技术约束

1. **依赖服务**：依赖 `IntentService` 的意图识别功能
2. **数据格式**：遵循项目统一的响应格式规范
3. **日志系统**：使用项目的 `log_decorator` 装饰器
4. **路由注册**：在 `chat_routes.py` 中注册新接口

## 验收标准

1. 接口能够正确接收用户问题并返回推荐问题列表
2. 类别1返回固定的3个问题
3. 类别2正确提取第2-4个结果的question字段
4. 错误情况下返回统一的错误响应
5. 日志记录完整，包含关键信息
6. 接口响应时间符合性能要求

## 后续扩展

1. 支持根据用户历史查询记录个性化推荐
2. 支持配置化管理类别1的固定问题列表
3. 支持更多意图类型的推荐策略
4. 支持推荐问题的相似度排序和去重
