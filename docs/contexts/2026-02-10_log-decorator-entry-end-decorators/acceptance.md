# 验收标准

## 功能验收

### 1. @log_entry 装饰器

#### 1.1 基本功能
- [ ] `@log_entry()` 可以成功装饰函数并执行
- [ ] 创建独立的日志文件 `logs/{entry_func}.log`
- [ ] 日志内容包含函数名、入参、出参、耗时
- [ ] 设置当前入口函数标记

#### 1.2 参数支持
- [ ] 支持 `print_args` 参数，控制是否记录入参
- [ ] 支持 `print_result` 参数，控制是否记录出参
- [ ] 支持 `print_duration` 参数，控制是否记录耗时
- [ ] 支持 `message` 参数，自定义开始日志文案
- [ ] 支持 `args_handler` 参数，自定义入参展示
- [ ] 支持 `result_handler` 参数，自定义出参展示
- [ ] 支持 `enable_mermaid` 参数，启用 Mermaid 记录器
- [ ] 支持 `force_mermaid` 参数，强制输出 Mermaid 文件
- [ ] 支持 `log_level` 参数，设置日志级别

#### 1.3 Mermaid 功能
- [ ] 当 `enable_mermaid=True` 时，初始化 Mermaid 记录器
- [ ] Mermaid 文件输出到 `logs/mermaid/{entry_func}/` 目录
- [ ] Mermaid 文件包含 ASCII 调用树、性能统计表、flowchart TD

#### 1.4 错误处理
- [ ] 装饰器内部错误不影响业务函数执行
- [ ] 发生异常时写入 `logs/error.log`
- [ ] 错误日志包含入口函数名、调用链路、错误信息、脱敏后的入参

---

### 2. @log_end 装饰器

#### 2.1 基本功能
- [ ] `@log_end()` 可以成功装饰函数并执行
- [ ] 记录函数的入参、出参、耗时
- [ ] 执行完毕后清理调用栈状态

#### 2.2 清理行为
- [ ] 重置调用深度计数器为 0
- [ ] 清除当前入口函数标记
- [ ] 关闭入口函数的独立日志处理器（如果存在）
- [ ] 触发 Mermaid 文件输出（如果启用）

#### 2.3 参数支持
- [ ] 支持所有 `@log` 装饰器的参数（与 @log_entry 一致）

#### 2.4 独立使用
- [ ] 可以单独使用 `@log_end`，不依赖 `@log_entry`
- [ ] 单独使用时不会报错或警告

---

### 3. 配对使用

#### 3.1 正常配对
- [ ] `@log_entry` 和 `@log_end` 可以配对使用
- [ ] 配对使用时，调用链日志完整记录
- [ ] `@log_end` 执行后，调用栈状态被正确清理

#### 3.2 嵌套场景
- [ ] 支持嵌套的 `@log_entry`，内层不会抢占外层入口
- [ ] 嵌套场景下，`@log_end` 只清理当前层级的状态

#### 3.3 异常场景
- [ ] 调用链中发生异常时，`@log_end` 仍能正确清理状态
- [ ] 异常信息被正确记录到 `logs/error.log`

---

### 4. 废弃 is_entry 参数

#### 4.1 代码变更
- [ ] `@log()` 装饰器不再接受 `is_entry` 参数
- [ ] 使用 `is_entry` 参数时抛出 TypeError 或参数错误

#### 4.2 文档更新
- [ ] README.md 中删除 `is_entry` 参数的说明
- [ ] 添加 `@log_entry` 和 `@log_end` 的使用文档
- [ ] 提供从 `@log(is_entry=True)` 迁移的指南

## 验收流程

1. **功能验收**：开发完成后，逐项检查功能验收标准
2. **测试验收**：运行所有测试用例，确保测试覆盖率和通过率
3. **性能验收**：进行性能测试，确保性能指标达标
4. **文档验收**：检查文档完整性和准确性
5. **迁移验收**：完成代码迁移，确保无回归问题
6. **最终评审**：与开发团队评审，确认所有验收标准已满足
