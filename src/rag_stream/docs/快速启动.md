# 🚀 快速启动指南

## 📋 前置要求

- Python 3.8+
- 网络连接到RAG服务 (192.168.188.3:8081)

## ⚡ 一键启动

### Windows用户
双击运行 `start_server.bat` 文件

### 其他系统用户
```bash
python start_server.py --reload
```

## 🔧 手动启动

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 启动服务
```bash
# 开发模式（自动重载）
python start_server.py --reload

# 生产模式
python start_server.py --workers 4

# 自定义端口
python start_server.py --port 8080
```

## 🎯 核心功能

### 用户会话绑定
- **自动会话管理**: 用户只需提供`user_id`，系统自动创建或复用会话
- **上下文连续性**: 同一用户在同一类别中的连续提问会使用相同会话
- **多类别独立**: 不同类别使用独立会话，避免混淆

### 使用示例
```bash
# 第一次提问 - 自动创建会话
curl -X POST "http://localhost:8000/api/laws" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "什么是安全生产法规？",
    "user_id": "user001"
  }'

# 第二次提问 - 自动使用同一会话，保持上下文
curl -X POST "http://localhost:8000/api/laws" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "这些法规有哪些具体要求？",
    "user_id": "user001"
  }'
```

## 🔍 常见问题

### Q: 服务启动失败？
A: 检查Python版本和依赖安装

### Q: 无法连接RAG服务？
A: 检查网络连接和RAG服务状态

### Q: 流式响应中断？
A: 检查超时配置和网络稳定性

### Q: 如何实现连续对话？
A: 每次请求都提供相同的`user_id`，系统会自动绑定会话

### Q: 会话会过期吗？
A: 是的，默认24小时后会话自动过期，可通过配置调整

## 💡 最佳实践

1. **始终提供user_id**: 确保每次请求都包含用户ID以实现会话绑定
2. **合理使用类别**: 不同类别的问题使用对应的接口
3. **监控会话状态**: 使用健康检查接口监控服务状态
4. **定期清理**: 系统会自动清理过期会话，无需手动干预
