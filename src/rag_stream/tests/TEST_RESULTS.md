## 关键功能验证

### ✅ JSON 格式转换
- **问题**: AI 返回 Python 格式的列表（单引号）`['污水处理厂']`
- **解决**: 增强 `_extract_json_from_text` 函数，使用 `ast.literal_eval()` 解析并转换为 JSON 格式
- **验证**: 所有测试用例都成功解析实体列表

### ✅ 返回格式统一
- **修改前**: 返回 `Dict[str, Any]`，包含 code、message、data 字段
- **修改后**: 返回 `List[Dict[str, str]]`，直接返回资源列表
- **验证**: 所有测试用例返回格式一致

### ✅ 错误处理
- **策略**: 错误时返回空列表 `[]`
- **日志**: 错误信息记录在日志文件中
- **验证**: 防护目标测试返回空列表，未抛出异常

### ✅ 意图分类
- **resource 意图**: 调用 `_get_solid_resource_instruction`，通过实体名称查询资源
  - 示例: "需要防护装备和防毒面具" → 提取实体 → 查询资源ID
- **requirement 意图**: 调用 `_query_resource_by_type`，通过 SQL 查询并 AI 筛选
  - 示例: "附近的避难场所" → SQL 查询 → AI 筛选 → 返回资源ID

### ✅ 多实体处理
- **测试用例**: "需要防护装备和防毒面具"
- **实体提取**: ['防护装备', '防毒面具']
- **返回结果**: 2 个资源ID
- **验证**: 成功处理多个实体并返回对应资源

## 日志分析

### 典型处理流程
1. 接收请求 → 验证事故ID
2. 查询事故数据 → 解析坐标信息
3. 并行调用 Dify:
   - 意图分类
   - 事故类型识别
   - 实体提取
4. 根据意图分发处理:
   - resource: 通过实体名称查询
   - requirement: 通过 SQL + AI 筛选
5. 返回资源ID列表

