# Entry 与 Marker 使用指南

## 何时使用 `@entry_trace`

- 把一次完整业务入口包成 `@entry_trace`，例如一次 API 请求、一次消息消费、一次批任务入口。
- 一个入口对应一份独立事件流和一份入口报告，便于单次问题复盘。
- 允许嵌套入口；内层入口会独立记录并在退出后恢复父入口上下文。

## 何时使用 `@trace`

- 在入口内部关键函数上使用 `@trace` 记录调用链与错误传播。
- 无入口上下文时，`@trace` 自动写入全局日志流（`global.events.jsonl`）。

## `marker` 关键点建议

- 在业务语义节点调用 `marker`，例如：库存检查开始、扣款提交前、回滚开始。
- `marker` 的 `kv` 建议只传最小关键上下文（如 `order_id`、`sku`）。
- 避免在高频循环里密集打 marker，以控制运行时开销。
