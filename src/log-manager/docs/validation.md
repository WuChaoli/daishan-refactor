# 长运行多进程验证记录

## 验证目标

- 验证入口与全局路由规则在多进程场景仍保持可解释性。
- 验证增量报告不依赖所有进程退出。
- 验证错误路径链与内存观测在持续运行中可持续输出。

## 验证步骤（建议）

1. 启动一个父进程，拉起多个工作子进程（模拟持续任务）。
2. 在子进程入口使用 `@entry_trace`，关键函数使用 `@trace`。
3. 每个子进程运行 3-5 分钟并周期触发 marker。
4. 人工注入一类可恢复错误与一类致命错误。
5. 观察报告目录是否持续产生快照并保留 `latest.txt`。

## 验收要点

- 可看到每个入口独立报告文件。
- 全局错误汇总按错误签名聚合并持续刷新。
- 报告包含调用链树与错误路径链。
- 报告包含进程内存与调用链内存信息（启用时）。

## 本次验证结果（2026-02-24）

- 使用 2 个子进程并行运行约 3 秒，入口函数与子函数均已打点。
- 已生成入口报告：2 份（每个子进程一个入口）。
- 已生成全局错误汇总：`global-error-summary/latest.txt`。
- 已观测到错误事件聚合与周期快照刷新。
- 验收结论：通过（满足持续运行下增量报告与错误汇总产出）。
