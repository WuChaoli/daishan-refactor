# 工作流程规范（精简版）

## 核心原则

**先流程后执行，先验证后结论。**

## 绝对禁令

- **禁止** 跳过 SKILL 检查直接开工
- **禁止** 未验证就声称“已完成”
- **禁止** 在主分支直接开发
- **禁止** 测试前先写生产代码（非明确例外）
- **禁止** 未做根因调查就盲目修复
- **禁止** 对可并行任务串行执行

## 必须遵守

- **必须** 在任务开始前判断并调用相关 SKILL（1% 规则）
- **必须** 多步骤任务维护 `update_plan`
- **必须** 每批改动后执行最小可证明验证
- **必须** 阻塞时立即停止并说明阻塞点
- **必须** 在响应中给出“已做什么 + 如何验证”

## 标准流程

1. 识别任务类型与目标
2. 检查并调用相关 SKILL
3. 制定计划并标记进行中步骤
4. 小步执行并持续验证
5. 汇总结果与风险
6. 请求反馈后进入下一批

## 阻塞处理

出现以下任一情况时，停止继续实现并先汇报：

- 缺少依赖或权限
- 测试持续失败且原因不明
- 需求存在歧义
- 连续 3 次尝试无有效进展

## 验证清单

- [ ] 已检查/调用相关 SKILL
- [ ] 已维护 `update_plan`
- [ ] 已执行实际验证命令
- [ ] 已记录验证结果与结论
- [ ] 阻塞时已明确说明原因与下一步
