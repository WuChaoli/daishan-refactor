# 开发执行规范（精简版）

## 核心原则

**隔离开发、测试驱动、根因修复。**

## 绝对禁令

- **禁止** 在 `main/master` 直接编码
- **禁止** 没有失败测试就写生产代码（TDD 例外需确认）
- **禁止** 多个猜测性修复一起提交
- **禁止** 跳过代码审查直接结束任务

## 必须遵守

### 1) 准备阶段

- **必须** 确认任务类型与相关 SKILL
- **必须** 创建隔离工作区（如 worktree）
- **必须** 验证基线测试可运行

### 2) 开发阶段（每个功能）

- **必须** 先写失败测试（RED）
- **必须** 用最小实现让测试通过（GREEN）
- **必须** 在绿色状态下重构（REFACTOR）
- **必须** 每轮仅做单一目标修改

### 3) 交付阶段

- **必须** 请求代码审查
- **必须** 修复 Critical/Important 问题
- **必须** 运行与改动直接相关的验证

## 调试流程（强制）

1. **根因调查**：稳定复现 + 定位边界
2. **模式对比**：与可工作实现对比差异
3. **假设验证**：一次只验证一个假设
4. **最小修复**：单点修复并回归验证

## TDD 例外（需用户明确许可）

- 一次性原型
- 生成代码
- 纯配置文件

## 验证清单

- [ ] 已隔离开发环境
- [ ] 已执行 RED→GREEN→REFACTOR
- [ ] 已完成代码审查与问题修复
- [ ] 已完成最小回归验证
