# 代码质量规范（精简版）

## 核心原则

**可读、可测、可维护、安全默认开启。**

## 绝对禁令

- **禁止** 硬编码密钥或凭证
- **禁止** 忽略错误与异常分支
- **禁止** 跳过输入验证直接入库/执行
- **禁止** 在生产代码保留调试输出
- **禁止** 未授权情况下绕过安全检查

## 必须遵守

### 代码结构

- **必须** 单一职责：函数只做一件事
- **必须** 控制复杂度：避免深层嵌套，优先提前返回
- **必须** 命名清晰：语义化命名优先于注释

### 健壮性

- **必须** 对外部调用做错误处理
- **必须** 对边界输入做校验（类型/格式/范围）
- **必须** 错误信息区分用户提示与技术日志

### 安全基线

- **必须** 使用环境变量管理敏感信息
- **必须** 防护注入/XSS/CSRF 等常见风险（按技术栈实现）
- **必须** 做鉴权与授权校验（涉及受保护资源时）

### 测试要求

- **必须** 为改动提供对应测试或验证步骤
- **必须** 覆盖关键路径与异常路径

## 推荐做法

- **应该** 小步提交，降低回归范围
- **应该** 提取重复逻辑，避免复制粘贴
- **应该** 使用项目既有格式化与风格规范

## 提交前检查清单

- [ ] 无硬编码密钥
- [ ] 输入已验证，错误已处理
- [ ] 安全边界检查已覆盖
- [ ] 相关测试/验证已通过
- [ ] 变更说明清晰可追踪
