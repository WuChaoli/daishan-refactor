## 主要发现

### 问题分析
1. **Type 1 完全失败**: 所有 6 个 Type 1 测试用例都被错误分类为 Type 2
   - 原因：岱山-数据库问题知识库的相似度分数普遍高于岱山-指令集
   - 优先级判断逻辑阈值 (0.6) 未能有效区分

2. **Type 3 大量误分类**: 18 个 Type 3 用例被错误分类为 Type 2
   - 主要涉及园区地址、负责人、发展历程等查询
   - 这些查询在数据库问题知识库中有高相似度匹配

3. **Type 2 表现最好**: 3 个用例中正确 2 个
   - "XX企业的危化品类目是什么？" → Type 2 ✓
   - "某化工厂的危险化学品有哪些？" → Type 2 ✓
   - "查询某公司的安全信息" → Type 1 ✗ (被误判为指令集)

## 建议改进

1. **调整优先级阈值**: 当前 priority_similarity_threshold=0.6 可能过高
2. **优化知识库内容**: 
   - 增强 Type 1 指令集知识库的覆盖面
   - 区分 Type 3 固定问题与 Type 2 数据库问题的边界
3. **引入分类器辅助**: IntentClassifier 可能帮助改善分类准确性
