## Key Decisions from CONTEXT.md

### Startup Flow
- Use startup event handler for initialization logic
- Database connection: try-retry mode, max 5 retries, 2s interval
- External services (Dify, RAGFlow): best-effort, poll ready state after startup
- Startup timeout: 30 seconds (matches Uvicorn timeout)

### Shutdown Behavior
- Respond to SIGTERM signal for graceful shutdown
- Shutdown order: stop accepting requests → finish active requests → close external connections → cleanup resources
- Graceful shutdown timeout: 30 seconds (Docker stop default)
- Force terminate remaining connections after timeout

### Resource Cleanup
- Explicitly close: database connection pool, HTTP client pool, Redis (if any)
- Cleanup order reverse of initialization (close higher-level resources first)
- All cleanup in lifespan shutdown

### Uvicorn Configuration
- Workers: 4 (based on container CPU limits)
- Graceful shutdown timeout: 30s
- Use lifespan protocol for startup/shutdown events
- Custom SIGTERM handler to log before shutdown

   - Wrap cleanup in try/finally to ensure it always runs
   - Add comprehensive logging at each phase

Implementation requirements:
