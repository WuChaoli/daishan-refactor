# Plan 12-01: Create Production Dockerfile

## Objective

Create a secure, optimized Dockerfile for the rag_stream service using multi-stage builds, non-root user execution, and proper signal handling for graceful shutdown.

## Requirements Addressed

- **DOCKER-01**: 多阶段 Dockerfile 构建（开发/生产分离）
  - 使用 uv 官方镜像优化构建层缓存
  - 分离构建依赖与运行时依赖
  - 最终镜像使用非 root 用户运行

## Success Criteria

1. ✅ Dockerfile builds successfully
2. ✅ Multi-stage build reduces final image size
3. ✅ Container runs as non-root user (UID ≥ 1000)
4. ✅ Graceful shutdown on SIGTERM works correctly
5. ✅ No secrets in image layers (verified via `docker history`)
6. ✅ Image size under 500MB

## Tasks

### Task 1: Create Dockerfile

**What:** Write production Dockerfile with multi-stage build

**Implementation Details:**
- Stage 1 (builder): Use `python:3.11-bookworm` for compilation tools
- Stage 2 (runtime): Use `python:3.11-slim-bookworm` for smaller footprint
- Install uv in builder stage
- Copy pyproject.toml and uv.lock first for cache optimization
- Install dependencies in builder, copy to runtime
- Create non-root user (appuser, UID 1000)
- Set working directory to /app
- Use exec form CMD for proper signal handling
- Expose ports 11027 and 11029
- Add HEALTHCHECK instruction

**Files to Create:**
- `Dockerfile`

**Verification:**
```bash
docker build -t rag_stream:test .
docker history rag_stream:test | grep -i secret  # Should be empty
docker run --rm rag_stream:test id  # Should show non-root user
```

### Task 2: Create .dockerignore

**What:** Exclude unnecessary and sensitive files from Docker context

**Implementation Details:**
- Ignore .git, __pycache__, .pytest_cache
- Ignore .env, .env.* files
- Ignore local virtual environments (.venv, venv)
- Ignore IDE files (.vscode, .idea)
- Ignore test artifacts and logs
- Ignore documentation (docs/, *.md except README)

**Files to Create:**
- `.dockerignore`

**Verification:**
```bash
docker build -t rag_stream:test . 2>&1 | head -20  # Should show small context
```

### Task 3: Verify Build and Test Graceful Shutdown

**What:** Build image and verify graceful shutdown behavior

**Implementation Details:**
1. Build Docker image
2. Run container with mounted config
3. Send SIGTERM and verify shutdown within 5 seconds
4. Check image layers for secrets
5. Verify image size

**Verification:**
```bash
# Build
docker build -t rag_stream:prod .

# Check size
docker images rag_stream:prod --format "{{.Size}}"

# Test graceful shutdown
docker run -d --name test_shutdown -v $(pwd)/config:/app/config rag_stream:prod
sleep 2
time docker stop test_shutdown  # Should complete in < 5s

# Verify non-root
docker run --rm rag_stream:prod whoami  # Should be appuser
```

## Technical Constraints

- **Base Image:** python:3.11-bookworm (builder), python:3.11-slim-bookworm (runtime)
- **User:** appuser, UID 1000
- **Workdir:** /app
- **Ports:** 11027, 11029
- **Signal Handling:** Exec form CMD for SIGTERM propagation
- **Healthcheck:** Interval 30s, timeout 10s, retries 3

## Dependencies

- Project must have pyproject.toml and uv.lock
- config.yaml must exist for testing

## Notes

- Use `uv pip install` for faster dependency resolution
- Builder stage can be larger; runtime stage should be minimal
- Consider using BuildKit for better layer caching
- Ensure uv.lock is committed before building
