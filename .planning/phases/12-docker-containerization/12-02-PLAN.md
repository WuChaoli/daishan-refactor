# Plan 12-02: Create Docker Compose Configurations

## Objective

Create docker-compose.yml for orchestration with proper service dependencies, volume mounts for configuration, and health checks.

## Requirements Addressed

- **DOCKER-02**: Docker Compose 生产编排配置
  - 创建 compose.yaml 定义服务、网络和卷
  - 配置服务依赖和健康检查
  - 支持环境变量注入敏感配置

## Success Criteria

1. ✅ docker-compose.yml is valid and can be parsed
2. ✅ Service starts with all configurations mounted
3. ✅ Health check passes during normal operation
4. ✅ Environment variables can override configuration
5. ✅ Service waits for dependencies (if applicable)

## Tasks

### Task 1: Create docker-compose.yml

**What:** Create production-ready Docker Compose configuration

**Implementation Details:**
- Define `rag_stream` service
- Mount config.yaml and .env as volumes
- Expose ports 11027:11027 and 11029:11029
- Set restart policy: unless-stopped
- Configure resource limits (optional)
- Use environment variables for sensitive config
- Define health check configuration

**Files to Create:**
- `docker-compose.yml`

**Configuration:**
```yaml
services:
  rag_stream:
    build: .
    container_name: rag_stream
    ports:
      - "11027:11027"
      - "11029:11029"
    volumes:
      - ./config/config.yaml:/app/config/config.yaml:ro
      - ./.env:/app/.env:ro
      - ./logs:/app/logs
    environment:
      - CONFIG_PATH=/app/config/config.yaml
      - ENV_FILE=/app/.env
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:11027/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
```

**Verification:**
```bash
docker-compose config  # Validate YAML syntax
docker-compose up -d   # Start services
docker-compose ps      # Check status
```

### Task 2: Create docker-compose.prod.yml (Optional Production Overrides)

**What:** Production-specific overrides for Docker Compose

**Implementation Details:**
- Resource limits (CPU, memory)
- Logging configuration (json-file driver with rotation)
- Security options (read-only root filesystem, no new privileges)
- Restart policy: always
- Remove development volumes

**Files to Create:**
- `docker-compose.prod.yml`

**Configuration:**
```yaml
services:
  rag_stream:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '0.5'
          memory: 512M
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
    read_only: true
    security_opt:
      - no-new-privileges:true
    tmpfs:
      - /tmp
```

**Verification:**
```bash
docker-compose -f docker-compose.yml -f docker-compose.prod.yml config
docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

## Technical Constraints

- **Network:** Default bridge (simpler for single-host)
- **Volumes:** Read-only for config, writable for logs
- **Restart Policy:** unless-stopped (dev), always (prod)
- **Health Check:** HTTP GET /health, 30s interval

## Dependencies

- Dockerfile from Plan 12-01 must exist and build successfully
- config.yaml must be present in ./config/
- .env file must be created from template

## Notes

- Use `docker-compose up --build` for development iterations
- Production deployment should use `docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d`
- Consider using Docker Swarm or Kubernetes for multi-host production
- Health check assumes /health endpoint exists (implemented in Phase 14)
