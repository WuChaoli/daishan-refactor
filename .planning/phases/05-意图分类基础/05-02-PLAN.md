在 IntentClassifier.classify 方法中完善降级逻辑：

1. 超时降级：
   - 使用 asyncio.wait_for 包装 LLM 调用，超时设置为 config.timeout（3 秒）
   - 捕获 asyncio.TimeoutError，返回 degraded=True 的默认结果
   - 记录 marker("classifier.timeout", {}, level="WARNING")

2. API 错误降级：
   - 捕获所有异常（Exception），返回 degraded=True 的默认结果
   - 记录 marker("classifier.error", {"error": str(e)}, level="ERROR")

3. 无效响应降级：
   - 如果 JSON 解析失败，返回 degraded=True 的默认结果
   - 如果 type_id 不在 {1, 2, 3} 中，返回 degraded=True 的默认结果
   - 如果 confidence 不在 0.0-1.0 范围内，返回 degraded=True 的默认结果
   - 记录 marker("classifier.invalid_response", {"reason": reason, "raw": raw_response[:100]}, level="WARNING")

4. 默认降级结果：
   - type_id = 0（或从 config 获取默认值）
   - confidence = 0.0
   - raw_response = ""
   - degraded = True

降级原则：任何异常情况都返回 degraded=True，不阻断主流程。
  </action>
