services:
  rag_stream:
    build:
      context: ../..
      dockerfile: src/rag_stream/Dockerfile
    # ... 标准配置
    
  e2e-test:
    image: python:3.12-slim
    volumes:
      - ../..:/app
      - ./reports:/reports

  <name>Task 4: 创建 GitHub Actions E2E 测试工作流</name>
  <files>.github/workflows/e2e-test.yml</files>
  <action>
创建 GitHub Actions 工作流配置，实现 CI/CD 环境中的自动 E2E 测试：

工作流触发条件：
- push: main 分支
- pull_request: main 分支
- workflow_dispatch: 手动触发

工作流步骤：
1. **Checkout**：检出代码
2. **Setup**：配置 Docker Buildx
3. **Create env file**：从 secrets 创建 .env 文件
   - DIFY_API_KEY
   - RAGFLOW_API_KEY
   - DATABASE_URL
   - 其他必要的环境变量
4. **Run E2E Tests**：
   - 使用 docker-compose.test.yml 启动服务
   - 等待测试完成
   - 捕获退出码
5. **Upload Report**：
   - 将 JSON 测试报告作为 artifact 上传
   - 设置保留期限（如 30 天）
6. **Cleanup**：
   - 停止并移除容器
   - 清理资源

关键配置：
```yaml
name: E2E Tests

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:

jobs:
  e2e:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Create .env file
        run: |
          echo "DIFY_API_KEY=${{ secrets.DIFY_API_KEY }}" >> .env
          echo "RAGFLOW_API_KEY=${{ secrets.RAGFLOW_API_KEY }}" >> .env
          # ... 其他环境变量
      
      - name: Run E2E Tests
        run: |
          docker compose -f tests/e2e/docker-compose.test.yml up --build --abort-on-container-exit
          EXIT_CODE=$?
          docker compose -f tests/e2e/docker-compose.test.yml down
          exit $EXIT_CODE
      
      - name: Upload Test Report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: e2e-test-report
          path: tests/e2e/reports/e2e_report.json
          retention-days: 30
```

优化考虑：
- 使用 Docker 层缓存加速构建
- 设置超时（如 15 分钟）防止测试挂起
- always() 确保即使测试失败也上传报告
- 在 PR 中显示测试摘要（可选未来增强）
